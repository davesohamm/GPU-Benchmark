PROBLEM IDENTIFIED:
-------------------
Both apps (CLI and GUI) were calling CUDA kernel launchers directly, even when
using OpenCL or DirectCompute backends. This caused immediate crashes.

ROOT CAUSE:
-----------
VectorAddBenchmark.cpp line 84:
    launchVectorAdd(...)  <-- This is a CUDA function!

When OpenCL or DirectCompute backend was selected, this CUDA function was
still called, causing crashes.

THE FIX:
--------
Created main_working.cpp that uses proper methods for each backend:
  - CUDA → launchVectorAdd()
  - OpenCL → CompileKernel() + ExecuteKernel()
  - DirectCompute → CompileShader() + DispatchShader()
